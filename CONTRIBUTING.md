# 贡献指南

感谢你关注 CraftTool！欢迎贡献你的力量。

## 如何贡献

### 报告问题

如果你发现模板中有 Bug 或缺少某些功能，请提交 Issue。

### 提交代码

1. Fork 感兴趣的模板仓库
2. 创建你的特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交你的更改 (`git commit -m 'Add amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建 Pull Request

### 代码规范

- 确保代码通过 ESLint/Prettier 检查
- 所有新功能需要包含测试
- 提交信息使用清晰简洁的描述

## 项目结构

每个模板仓库都遵循统一的项目结构：

```
├── .github/workflows/     # CI/CD 配置
├── src/                  # 源代码
├── test/                 # 测试文件
├── README.md             # 项目文档
└── package.json          # 依赖配置
```

## 许可证

通过贡献代码，你同意你的贡献使用 MIT 许可证。
